# Professional Services Variant

Extends the core dann-specs principles for client-facing professional work (medicine, law, accounting, design, architecture, trades, etc.).

**Core principles still fully apply** (see root AGENTS.md and relevant roles/).

## Domain-Specific Emphasis
- Prioritize client/patient outcomes, ethics, compliance, and clear communication.
- Use role-specific guidance from the roles/ directory (e.g., physician.md, lawyer.md, accountant.md).
- Always make assumptions, tradeoffs, and verification explicit for the client.
- Document decisions and maintain auditability/reproducibility appropriate to the field.
- Verification preference: Domain-appropriate (clinical guidelines, legal precedents, financial reconciliations, design reviews, functional tests, measurements, or client sign-off).

Combine with the matching role file for best results.
