# Variante infra-devops

Extiende los principios fundamentales de dann-specs con enfoque de dominio.

**Los principios fundamentales siguen aplicando** (ver AGENTS.md raíz).

## Énfasis del Dominio
- Infraestructura como código y control de versiones para toda configuración.
- Priorizar claridad, reproducibilidad y automatización.
- Usar métodos de verificación apropiados al dominio (pruebas de integración, comprobaciones de健康, pruebas de humo).
- Documentar planes de rollback y procedimientos de recuperación.
- Implementar el principio de mínimo privilegio para acceso a infraestructura.
- Auditar configuraciones de seguridad y superficies de exposición.
